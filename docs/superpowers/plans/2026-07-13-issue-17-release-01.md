# Issue 17 NestJS 0.1 Release Implementation Plan

> **For Codex:** Complete and merge the release preparation independently; do not mark the issue complete until the public npm packages are verified.

**Goal:** Prove `activitylog-core` and `activitylog-nestjs` as installed consumer artifacts and prepare their two-phase Changesets release at version 0.1.0 with provenance.

**Architecture:** A standalone Node script builds tarballs, creates a temporary consumer outside the workspace, installs the tarballs plus exact Nest peers, boots a real Nest application context, resolves `ActivityLogService`, and proves the leaf package shares core AsyncLocalStorage state. CI and release run this smoke after build. Changesets target only core and NestJS at minor version; NextJS remains unpublished until 0.2.

**Tech Stack:** npm workspaces/pack, NestJS 11, Changesets, GitHub Actions, Node 20.

---

### Task 1: Build the external Nest consumer smoke

**Files:**
- Create: `scripts/smoke-nestjs-consumer.cjs`
- Modify: `package.json`

Pack core and NestJS into a temporary directory, install them into an isolated consumer with exact peers, boot `NestFactory.createApplicationContext`, and log an Activity under `runWithContext`. Assert the Activity, causer, event, and package versions from the installed artifacts. Always clean the temporary consumer.

### Task 2: Put the smoke in CI and release

**Files:**
- Modify: `.github/workflows/ci.yml`
- Modify: `.github/workflows/release.yml`

Run the consumer smoke after the monorepo build in both workflows so publishing cannot bypass the artifact-level check.

### Task 3: Prepare exactly the 0.1 packages

**Files:**
- Modify: `.changeset/initial-bootstrap.md`
- Modify: `packages/activitylog-nestjs/package.json`
- Modify: `packages/activitylog-nextjs/package.json`
- Create: `.changeset/nestjs-typeorm-01.md`
- Modify: `docs/NEXT-STEPS.md`

Target minor releases for core and NestJS only, keep NextJS at 0.0.0/unpublished, and make both leaf dependency ranges accept core 0.1. Confirm the Changesets status reports core and NestJS 0.1.0 without a NextJS release.

### Task 4: Validate, merge preparation, then release externally

Run lint, typecheck, coverage, build, smoke, pack dry-runs, audit, and diff checks. Open and merge the preparation PR only with green CI. Then use the generated `changeset-release/main` commit to open the version PR manually if repository Actions permissions still prevent the bot PR. Do not merge the version PR or close #17 until npm authentication is configured; after it is available, merge, wait for release CI, and verify both public package versions and provenance.
